Q01 What is Flutter and why choose it over other frameworks?
Basic▼
Answer

Flutter is Google's open-source UI toolkit for building natively compiled, beautiful applications for mobile, web, and desktop from a single Dart codebase.

  • Single codebase — iOS, Android, Web, macOS, Windows, Linux from one project
  • Hot Reload — See changes instantly without losing app state
  • Own rendering engine — Skia/Impeller draws every pixel; no platform UI components needed
  • 60/120fps — Dart compiles to native ARM; no JavaScript bridge bottleneck
  • Rich widgets — Material (Android) and Cupertino (iOS) widget libraries built-in
  • Great for MVPs — Faster time to market, lower development cost
Key difference: Unlike React Native which bridges to native components, Flutter draws its own UI entirely — giving pixel-perfect consistency across platforms.

Q03 Explain the StatefulWidget lifecycle in order.
Basic▼
Answer
  1. createState() — Framework calls this once; returns the State object.
  2. initState() — First method on State. Initialize controllers, subscribe to streams. Call super.initState() first.
  3. didChangeDependencies() — Called after initState AND whenever an InheritedWidget dependency changes.
  4. build() — Returns the widget tree. Called on every setState, didUpdateWidget, or didChangeDependencies.
  5. didUpdateWidget(old) — Parent rebuilt with new configuration for this widget.
  6. setState(fn) — Schedules a rebuild; fn should be synchronous.
  7. deactivate() — Widget removed from tree temporarily (might be re-inserted).
  8. dispose() — Widget permanently removed. Release all resources: controllers, streams, timers.
Rule of thumb: Initialize in initState, clean up in dispose. Never use BuildContext after dispose.
Q04 What is BuildContext and what is it used for?
Basic▼
Answer

BuildContext is a reference to the widget's location in the widget tree. It lets you:

  • Access ancestor data: Theme.of(context), MediaQuery.of(context)
  • Navigate: Navigator.of(context).push(...)
  • Read state management: context.read<MyBloc>(), Provider.of<T>(context)
  • Show overlays: showDialog(context: context, ...)
Warning: Never store a BuildContext across async gaps without checking mounted first, or you'll get a "use of BuildContext across async gap" error.

Q07 What is the difference between Expanded and Flexible?
Basic▼
Answer

Expanded forces the child to fill all remaining space in the main axis (FlexFit.tight).
Flexible allows the child to be smaller than the allocated space (FlexFit.loose by default — WRAP_CONTENT behavior).

In fact, Expanded is just shorthand for Flexible(fit: FlexFit.tight).

Row(children: [
  Expanded(child: Text('Takes ALL remaining space')),   // fills max
  Flexible(child: Text('Takes ONLY what it needs')),  // wraps content
])
Q08 What is pubspec.yaml and what does it contain?
Basic▼
Answer

The project manifest file in YAML format. It defines:

  • name, description, version — App identity
  • environment — Dart SDK constraint (sdk: ">=3.0.0 <4.0.0")
  • dependencies — Packages required in production builds
  • dev_dependencies — Packages only for development/testing (e.g., mockito, flutter_test)
  • flutter.assets — Image, font, and other static asset paths
Difference: dependencies are bundled into the release APK/IPA. dev_dependencies are stripped out — they never reach production.
Q09 What is the difference between packages and plugins in Flutter?
Basic▼
Answer
  • Package — Contains only Dart code. No native code. (e.g., equatable, dartz)
  • Plugin — Contains Dart code + native code (Kotlin/Swift). Accesses platform APIs. (e.g., camera, firebase_core)

A package can depend on plugins and still be classified as a package.

Q10 What are MaterialApp vs CupertinoApp?
Basic▼

Q11 What is ListView vs ListView.builder?
Basic▼
Answer
  • ListView — Renders all children at once (eager). Good for small fixed lists (<20 items).
  • ListView.builder — Lazily renders only visible items on demand. Essential for large/infinite lists. O(1) memory regardless of list size.
  • ListView.separated — Same as builder but adds separators between items.

Q12 What are Keys in Flutter and when do you use them?
Basic▼
Answer

Keys help Flutter uniquely identify widgets across rebuilds, preserving state when widgets move in the tree.

  • ValueKey — Keyed by a value: ValueKey('user-42')
  • ObjectKey — Keyed by object identity
  • UniqueKey() — Always different; forces recreation
  • GlobalKey — Access a widget's state/RenderBox from anywhere
  • PageStorageKey — Preserve scroll positions across navigation
When you MUST use keys: When reordering a list of StatefulWidgets (e.g., a draggable todo list). Without keys, Flutter may assign wrong state to wrong widget.

Q13 Explain Future, Stream, async, await, async*, and yield in Dart.
Intermediate▼
Answer
  • Future<T> — A single async value resolved once. Either completes with value or error.
  • Stream<T> — A sequence of async events over time. Multiple values, each potentially with an error.
  • async — Marks a function as asynchronous; return type becomes Future automatically.
  • await — Pauses the current async function until the Future resolves.
  • async* — Generator function that produces a Stream. Uses yield to emit values.
  • yield — Emits a single value into the Stream from an async* function.
  • yield* — Delegates to another Stream (emits all values from it).
// Future
Future<String> fetchUser(int id) async {
  final res = await http.get(Uri.parse('/api/user/$id'));
  return res.body;
}

// Stream with async*
Stream<int> countdown(int n) async* {
  for (int i = n; i >= 0; i--) {
    await Future.delayed(Duration(seconds: 1));
    yield i;
  }
}

Q16 What is a Mixin in Dart and how is it different from inheritance?
Intermediate▼
Answer

A mixin is a way to reuse code across multiple class hierarchies without subclassing. It "plugs in" methods/properties using the with keyword. Solves the diamond inheritance problem.

Key differences from inheritance: a class can use multiple mixins (with A, B, C) but only extend one class.

mixin Logger {
  void log(String msg) => print('[LOG] $msg');
}

mixin Validator {
  bool isEmail(String s) => s.contains('@');
}

class AuthService with Logger, Validator {
  void login(String email) {
    if (isEmail(email)) log('Logging in: $email');
  }
}
Flutter example: SingleTickerProviderStateMixin is used to make a State an AnimationController ticker provider.

Q20 What are Iterables and collections in Dart?

Q23 What are Runes and Symbols in Dart?
Intermediate▼
Answer
  • Runes — A String is UTF-16 code units. string.runes exposes the actual Unicode code points (integers), allowing correct iteration of emoji and non-BMP characters. '😀'.runes.first → 128512
  • Symbol — Represents an identifier/operator in a Dart program. Created with #name syntax. Primarily used in reflection (dart:mirrors) and code generation. Symbols are compile-time constants and canonicalized, so #print == #print is true.
print('Héllo'.runes.length);   // 5 — correct Unicode length
print('Héllo'.length);        // 6 — UTF-16 code unit count

var sym = #myIdentifier;   // Symbol
print(sym.toString());       // Symbol("myIdentifier")

Q25 What are OOP concepts in Dart? (Encapsulation, Inheritance, Polymorphism, Abstraction)
Intermediate▼
Answer
  • Encapsulation — Bundle data + methods in a class. Use _ prefix for private members. Controls access.
  • Inheritance — extends creates an is-a relationship. Single inheritance in Dart (use mixins for multiple).
  • Polymorphism — Runtime polymorphism via method overriding (@override). Dart doesn't support method overloading.
  • Abstraction — abstract class defines interface without full implementation. implements forces all method implementations.
abstract class Shape {
  double area(); // must implement
  void describe() => print('Area: ${area()}'); // concrete
}

class Circle extends Shape {
  final double r;
  Circle(this.r);
  @override double area() => 3.14 * r * r;
}
Q26 What is the factory constructor in Dart?
Intermediate▼
Answer

A factory constructor can return an existing instance (not always a new one) and can return a subtype. Used for: singletons, caching, named constructors for JSON, and polymorphic creation.

// Singleton
class Database {
  static final Database _instance = Database._internal();
  Database._internal();
  factory Database() => _instance;
}

// JSON deserialization
class User {
  final String name; final int age;
  User({required this.name, required this.age});
  factory User.fromJson(Map<String, dynamic> j) =>
      User(name: j['name'], age: j['age']);
}
Q27 What is toString() and hashCode in a Dart class?
Intermediate▼
Answer
  • toString() — Returns a human-readable string of the object. Override for meaningful debug output in print() and debugger.
  • hashCode — Returns an integer used in HashMap and HashSet lookups. Must be consistent with ==: if a == b then a.hashCode == b.hashCode.
  • == operator — By default compares object identity. Override to compare by value.
class Point {
  final int x, y;
  const Point(this.x, this.y);

  @override
  String toString() => 'Point($x, $y)';

  @override
  int get hashCode => Object.hash(x, y);

  @override
  bool operator ==(Object other) =>
      other is Point && other.x == x && other.y == y;
}

Q29 What are Flutter's three render trees? (Widget, Element, RenderObject)
Intermediate▼
Answer
  • Widget Tree — Configuration/blueprint. Immutable, rebuilt frequently. Cheap because widgets are just Dart objects describing the UI.
  • Element Tree — The live instantiation of the widget tree. Persists across rebuilds. Each element holds a reference to its widget and (for StatefulWidgets) to the State object. This is Flutter's reconciliation layer.
  • RenderObject Tree — Handles layout, painting, and hit-testing. Most expensive to rebuild. Flutter only updates this when necessary. Stores actual dimensions, paint commands, etc.
Why this matters for performance: Rebuilding a widget is cheap (just creates a new object). Flutter's element diffing decides whether to update the underlying RenderObject — which is the expensive part.
Q30 What is InheritedWidget and how does it differ from Provider?
Intermediate▼
Answer

InheritedWidget is Flutter's built-in mechanism for passing data down the tree efficiently. Descendants call context.dependOnInheritedWidgetOfExactType<T>() and are rebuilt only when it changes. Theme, MediaQuery, and Navigator are built on it.

Provider is a wrapper over InheritedWidget with a simpler API, automatic disposal, lazy initialization, and support for ChangeNotifier, Stream, and Future.

Mental model: Provider is InheritedWidget with a better developer experience. Under the hood, Provider.of<T>(context) calls the same InheritedWidget lookup.

Q31 What is the BLoC pattern? How does it work?
Intermediate▼
Answer

BLoC (Business Logic Component) separates UI from business logic using a unidirectional data flow:

UI → dispatches Event → Bloc → emits State → UI rebuilds

  • Event — User action / intent (e.g., LoginPressed(email, password))
  • Bloc — Receives events via on<Event> handlers, processes them, emits states
  • State — Snapshot of UI data (e.g., LoginLoading, LoginSuccess(user), LoginError(msg))
  • Cubit — Simplified BLoC without events. Just methods that call emit().
// Cubit (simpler)
class CounterCubit extends Cubit<int> {
  CounterCubit() : super(0);
  void increment() => emit(state + 1);
  void decrement() => emit(state - 1);
}

// Full Bloc
class AuthBloc extends Bloc<AuthEvent, AuthState> {
  AuthBloc() : super(AuthInitial()) {
    on<LoginPressed>(_onLogin);
  }
  Future<void> _onLogin(LoginPressed e, Emitter<AuthState> emit) async {
    emit(AuthLoading());
    try {
      final user = await _repo.login(e.email, e.password);
      emit(AuthSuccess(user));
    } catch (err) { emit(AuthError(err.toString())); }
  }
}
Q32 What are all the BLoC widgets? (BlocProvider, BlocBuilder, BlocListener, BlocConsumer, BlocSelector)
Intermediate▼
Answer
  • BlocProvider — Creates and provides a Bloc to the widget subtree. Use at the top of a feature/page.
  • MultiBlocProvider — Provides multiple blocs without nesting.
  • BlocBuilder — Rebuilds UI on state changes. buildWhen controls when to rebuild.
  • BlocListener — Listens to states for side effects (navigation, snackbars, dialogs). Does NOT rebuild UI. listenWhen controls when to trigger.
  • BlocConsumer — Combines BlocBuilder + BlocListener in one widget. Use when you need both UI rebuild AND side effects.
  • BlocSelector — Rebuilds only when a specific derived value changes (fine-grained optimization).
  • RepositoryProvider — Provides a repository (or service) to the widget tree. Similar to BlocProvider but for non-Bloc objects.
Q33 What is buildWhen and listenWhen in BLoC?
Intermediate▼
Answer

Both accept a function (previousState, currentState) → bool. Return false to skip the rebuild/listener call.

BlocBuilder<CartBloc, CartState>(
  // Only rebuild when item count actually changes
  buildWhen: (prev, curr) => prev.itemCount != curr.itemCount,
  builder: (ctx, state) => Badge(count: state.itemCount),
)

BlocListener<AuthBloc, AuthState>(
  // Only navigate to home on first AuthSuccess
  listenWhen: (prev, curr) => curr is AuthSuccess && prev is! AuthSuccess,
  listener: (ctx, state) => ctx.go('/home'),
  child: LoginForm(),
)
Performance win: Use buildWhen to prevent rebuilding expensive widgets when unrelated state properties change.

Q35 What is Equatable and why use it in BLoC?
Intermediate▼
Answer

Dart compares objects by reference by default. BLoC only emits a new state if it differs from the previous one (newState != oldState). Without value equality, two states with identical data appear different — causing duplicate rebuilds.

Equatable overrides == and hashCode based on a props list, so two state objects with the same data are considered equal.

class ProductLoaded extends ProductState with EquatableMixin {
  final List<Product> products;
  const ProductLoaded(this.products);
  @override
  List<Object> get props => [products];
}

Q37 What is MVVM in Flutter?
Intermediate▼
Answer
  • Model — Data layer: entities, repositories, API calls
  • ViewModel — Holds and transforms UI state; exposes streams or ValueNotifiers; handles business logic. Implemented with ChangeNotifier, Riverpod Notifier, or Cubit.
  • View — Flutter widgets. Observes ViewModel, calls its methods. Zero business logic.
BLoC as MVVM: The Bloc serves as the ViewModel — it receives events (user actions) and exposes states (UI data). BlocProvider injects it into the View layer.
Q38 What is Hive database in Flutter?
Intermediate▼
Answer

Hive is a fast, lightweight, pure-Dart key-value database for local storage. No native dependencies.

  • Box — A named, typed container. Hive.openBox('settings')
  • TypeAdapter — Serialize/deserialize custom Dart objects. Auto-generated with hive_generator.
  • HiveObject — Base class for stored models; gives key, save(), delete() methods.
// Open box and use it

box.put('currentUser', User(name: 'Ali'));
final user = box.get('currentUser'); // User?
Security: Use HiveAesCipher with a key from flutter_secure_storage to encrypt sensitive boxes.
Q39 What is Ephemeral state vs App state?
Intermediate▼
Answer
  • Ephemeral (local) state — State that lives inside a single widget and doesn't need to be shared. Managed with setState(). E.g., current tab index, animation progress, form field focus.
  • App (shared) state — State used by multiple widgets across the app. Managed with Provider, BLoC, Riverpod, GetX, etc. E.g., cart items, auth status, user profile.
Rule: Start with setState(). Lift to app state only when multiple unrelated widgets need the same data.

MultiBlocProvider(

Q41 What is Clean Architecture in Flutter?
Advanced▼
Answer

Clean Architecture divides the app into three layers with strict inner-to-outer dependency rules:

  • Domain Layer (innermost) — Business logic only. Pure Dart. Contains: Entities, Use Cases, Repository Interfaces. Has zero Flutter/HTTP/database imports.
  • Data Layer — Implements Domain interfaces. Contains: API models (DTOs), data sources (Retrofit/Dio), repository implementations, local DB.
  • Presentation Layer (outermost) — Flutter widgets, BLoC/Cubit. Depends on Domain use cases, never on Data directly.
lib/
├── core/                  # DI, network client, theme, constants
├── features/
│   └── auth/
│       ├── domain/
│       │   ├── entities/user.dart
│       │   ├── repositories/auth_repository.dart   # interface
│       │   └── usecases/login_usecase.dart
│       ├── data/
│       │   ├── models/user_model.dart
│       │   ├── sources/auth_remote_source.dart
│       │   └── repositories/auth_repository_impl.dart
│       └── presentation/
│           ├── bloc/auth_bloc.dart

└── main.dart
Q42 What are the SOLID Principles?
Advanced▼
Answer
  • S — Single Responsibility — A class/function has one reason to change. AuthBloc handles auth only; CartBloc handles cart only.
  • O — Open/Closed — Open for extension, closed for modification. Add behavior via new subclasses/mixins, not by editing existing code.
  • L — Liskov Substitution — Subclasses must be usable wherever the base class is expected, without breaking the program.
  • I — Interface Segregation — Many small, specific interfaces are better than one large general interface. Don't force a class to implement methods it doesn't need.
  • D — Dependency Inversion — High-level modules depend on abstractions (interfaces), not concretions. The AuthBloc depends on AuthRepository (abstract), not AuthRepositoryImpl.
Q43 What is Dependency Injection in Flutter and how do you implement it?
Advanced▼
Answer

DI is a pattern where objects receive their dependencies from outside instead of creating them. This makes code testable (swap real implementations with mocks) and decoupled.

Common Flutter DI approaches:

  • Constructor injection — Simplest. Pass dependencies via constructor.
  • get_it — Service locator. Register singletons/factories once, access anywhere. Fastest runtime lookup.
  • Provider / Riverpod — DI via the widget tree. Scoped and reactive.
  • injectable — Code generation for get_it, similar to Dagger on Android.
// get_it setup

  sl.registerLazySingleton<Dio>(() => Dio(BaseOptions(baseUrl: kBaseUrl)));

Q44 How do you optimize Flutter app performance?
Advanced▼
Answer
  • const constructors — Prevents widget recreation when parent rebuilds. const Text('Static') is never rebuilt.
  • Narrow setState scope — Extract animated/updating parts to child widgets so only they rebuild.

  • buildWhen / select() — Prevent BLoC/Provider rebuilds when unrelated state changes.

Q45 How do you secure a Flutter app? (JWT, SSL Pinning, storage)
Advanced▼
Answer
  • JWT Storage — Always use flutter_secure_storage (Keychain on iOS, EncryptedSharedPreferences on Android). Never localStorage, SharedPreferences, or hardcode.
  • Token security — Short-lived access tokens (15min) + longer refresh tokens. Rotate on use. Revocation list on server.
  • SSL/Certificate Pinning — Embed server certificate SHA256 fingerprint. Reject any connection whose cert doesn't match. Prevents MITM. Implement via Dio interceptor.
  • Code obfuscation — flutter build apk --obfuscate --split-debug-info=build/. Hides class/method names from reverse engineering.
  • API key protection — Never hardcode API keys. Use env files, obfuscation, or a backend proxy.
  • Root/Jailbreak detection — flutter_jailbreak_detection package.
  • Background screenshot protection — secure_application package.
  • HMAC — Hash-based MAC for message integrity verification (both parties share secret key).
  • RSA — Asymmetric key encryption for initial key exchange and digital signatures.
Q46 What are Platform Channels and how do you write native code?
Advanced▼
Answer

Platform Channels let Dart communicate with Kotlin/Java (Android) or Swift/Objective-C (iOS) via message passing.

  • MethodChannel — Request/response. Call native method from Dart.
  • EventChannel — Native pushes a stream of events to Dart (e.g., sensor data).
  • BasicMessageChannel — Bi-directional arbitrary messages.
// Dart side
const _ch = MethodChannel('com.app/battery');
Future<int> getBattery() => _ch.invokeMethod<int>('getBatteryLevel').then((v) => v!);

// Kotlin (Android) — MainActivity.kt
MethodChannel(flutterEngine.dartExecutor.binaryMessenger, "com.app/battery")
    .setMethodCallHandler { call, result ->
        when (call.method) {
            "getBatteryLevel" -> result.success(getBatteryLevel())
            else -> result.notImplemented()
        }
    }

Q48 How do you implement offline mode in a Flutter app?

Q49 What is Deep Linking and Dynamic Linking?
Advanced▼
Answer

Q54 What is Push Notification and how is it implemented in Flutter?
Advanced▼
Answer

Push notifications alert users to app events when the app is in background or killed. Implementation via Firebase Cloud Messaging (FCM):

  1. Add firebase_messaging package
  2. Initialize Firebase (Firebase.initializeApp())
  3. Request permissions: await messaging.requestPermission()
  4. Get FCM token: await messaging.getToken() → send to your backend
  5. Handle messages in 3 states:
// Foreground (app open)
FirebaseMessaging.onMessage.listen((msg) => showLocalNotification(msg));

// Background tap (app in background)
FirebaseMessaging.onMessageOpenedApp.listen((msg) => navigateTo(msg));

// Terminated (app closed)
FirebaseMessaging.instance.getInitialMessage().then((msg) {
  if (msg != null) navigateTo(msg);
});
Use flutter_local_notifications to show custom notification UI when the app is in the foreground.

Q58 What is the RepaintBoundary widget?
Advanced▼
Answer

RepaintBoundary creates a separate GPU compositing layer for its child. When the child repaints, Flutter composites only that layer — the rest of the screen is not touched.

When to use:

  • Complex animations (scrolling list items, particle effects)
  • Charts or custom painters that update frequently
  • Widgets that are expensive to paint but rarely change their layout
Caution: Each RepaintBoundary allocates a texture on the GPU. Overuse increases memory pressure. Always profile first with Flutter DevTools "Highlight Repaints" toggle.

Q59 What is Concurrency in Dart? How does the Event Loop work?
Expert▼
Answer

Dart is single-threaded. Concurrency is achieved via an event loop, not threads. The event loop has two queues:

  • Microtask Queue — Highest priority. Processed completely before moving to event queue. Used by Future.microtask(), scheduleMicrotask().
  • Event Queue — Regular async events: timer callbacks, I/O completions, Future.delayed(), user input.

Loop: Execute all microtasks → execute ONE event → repeat.

True parallelism (multiple CPU cores) requires Isolates, which each have their own event loop and heap, and communicate only via message passing.

print('1 - sync');
Future.microtask(() => print('3 - microtask'));
Future(() => print('4 - event queue'));
print('2 - sync');
// Output: 1, 2, 3, 4
Q60 What are Isolates and when do you use them?
Expert▼
Answer

Isolates are independent execution contexts with their own memory heap and event loop. They communicate via message passing (SendPort/ReceivePort) — no shared memory. This eliminates race conditions.

Use when: CPU-intensive work that would block the UI thread >16ms (1 frame): JSON parsing, image processing, encryption, compression.

// Simple: use compute() — spawns isolate, runs function, returns result
final users = await compute(_parseUsers, jsonString);

List<User> _parseUsers(String json) => // top-level or static function!
    (jsonDecode(json) as List).map(User.fromJson).toList();

// Advanced: long-lived isolate with bidirectional communication
final port = ReceivePort();
await Isolate.spawn(_workerEntry, port.sendPort);
final workerSendPort = await port.first as SendPort;
Q61 What is FFI in Dart?
Expert▼
Answer

FFI (Foreign Function Interface) allows Dart to directly call native C/C++ libraries without platform channels. Faster than method channels for performance-critical native code.

import 'dart:ffi';
import 'dart:io';

final lib = DynamicLibrary.open(
  Platform.isAndroid ? 'libcrypto.so' : 'libcrypto.dylib'
);

// Bind to a C function: int SHA256_Init(SHA256_CTX *c)
typedef SHA256Native = Int32 Function(Pointer);
typedef SHA256Dart   = int   Function(Pointer);
final sha256init = lib.lookupFunction<SHA256Native, SHA256Dart>('SHA256_Init');

Use FFI for: crypto libraries, image codecs, existing C/C++ SDKs, game engines.

Q62 What is a Sealed Class in Dart 3?
Expert▼
Answer

Sealed classes restrict their subclasses to a finite known set within the same library. The Dart compiler performs exhaustive pattern matching — it's a compile error if you miss a case in a switch.

sealed class NetworkState {}
class Loading   extends NetworkState {}
class Success<T> extends NetworkState { final T data; Success(this.data); }
class Failure   extends NetworkState { final String err; Failure(this.err); }

// Compiler error if any case is missing:
switch (state) {
  case Loading():    return CircularProgressIndicator();
  case Success(:var data): return Text('$data');
  case Failure(:var err):  return Text('Error: $err');
}

Q65 What is tree shaking in Flutter?
Expert▼
Answer

Tree shaking is dead code elimination during AOT compilation. The Dart compiler traces all reachable code from the app entry point and removes everything else from the final binary.

This is why compile-time constants like kReleaseMode are powerful — code in unreachable branches is completely removed:

if (kDebugMode) {
  print('Debug info'); // ← REMOVED in release build
}

Icons are also tree-shaken — only the specific icon code points used in your app are included, not the entire Material icon font.

Q66 🖊️ CODING: Sort an array without predefined sort methods.
Expert▼
Answer — Multiple Algorithms
// ─── Bubble Sort — O(n²) time | O(1) space ───
List<int> bubbleSort(List<int> arr) {
  final a = [...arr]; // copy
  for (var i = 0; i < a.length - 1; i++) {
    for (var j = 0; j < a.length - i - 1; j++) {
      if (a[j] > a[j+1]) { final t = a[j]; a[j] = a[j+1]; a[j+1] = t; }
    }
  }
  return a;
}

// ─── Quick Sort — O(n log n) avg | O(log n) space ───
List<int> quickSort(List<int> arr) {
  if (arr.length <= 1) return arr;
  final pivot = arr[arr.length ~/ 2];
  return [
    ...quickSort(arr.where((x) => x < pivot).toList()),
    ...arr.where((x) => x == pivot),
    ...quickSort(arr.where((x) => x > pivot).toList()),
  ];
}

// Test
print(quickSort([5,1,9,3,7])); // [1, 3, 5, 7, 9]

Q69 What is SAML / SSO Authentication?
Expert▼
Answer

SAML (Security Assertion Markup Language) is an XML-based protocol for Single Sign-On (SSO). It lets a user log in once to an Identity Provider (IdP) and gain access to multiple Service Providers (SP) without re-authenticating.

  • Identity Provider (IdP) — Authenticates the user (e.g., Okta, Azure AD, Google Workspace)
  • Service Provider (SP) — The app the user wants to access
  • SAML Assertion — Signed XML document containing user identity + attributes sent from IdP to SP

In mobile apps, SSO is typically implemented via OAuth 2.0 / OpenID Connect (OIDC) using packages like flutter_appauth.
